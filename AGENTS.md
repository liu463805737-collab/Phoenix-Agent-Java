# Phoenix Agent

Multi-module Maven project (Java 21). An NL2SQL state-graph engine built on Spring AI Alibaba.

## Modules

| Module | Tier | Notes |
|--------|------|-------|
| `phoenix-parent` | BOM | Dependency & plugin management |
| `phoenix-data` | App | **Main application** — entrypoint, graph workflow, controllers, services, MyBatis mappers |
| `phoenix-agent` | Sub-domain | Three-tier: `-api` / `-core` (MyBatis-Flex) / `-rest` |
| `phoenix-admin`, `phoenix-privilege`, `phoenix-platform` | Sub-domains | Three-tier: `-api` / `-core` / `-rest` |
| `phoenix-common` | Shared | `-api` (MyBatis-Flex starter), `-core` (Redis/JetCache/Caffeine), `-rest` |
| `phoenix-tool` | Utility | Shared utilities |
| `phoenix-codegen` | Codegen | MyBatis-Flex codegen |
| `phoenix-flink` | Flink | Three-tier: `-api` / `-core` / `-server` |

## Config & Setup

- **No `application.yml` / `application.properties`** — zero resource files. All config via env vars or `@ConfigurationProperties` defaults.
- Properties prefix: `spring.ai.phoenix.data-agent.*` (defined in `DataAgentProperties.java`).
- **No tests exist** — `src/test/` is absent everywhere despite test deps in POM.
- `-parameters` compiler arg enabled for MyBatis parameter name reflection.
- `flatten-maven-plugin` is **commented out** in parent POM. `${revision}` is used for versions but not resolved by flatten.
- No CI/CD, no Dockerfile, no Makefile, no pre-commit hooks.

## Architecture

- **Entrypoint**: `phoenix-data/.../DataAgentApplication.java` — `@SpringBootApplication` + `@EnableScheduling`
- **Health check**: `GET /echo/ok` (`EchoController`)
- **Main streaming endpoint**: `GET /api/stream/search` (`GraphController`) — returns SSE flux via `Sinks.Many`
- **REST API base**: controllers under `com.phoenix.data.controller`, all mapped under `/api` with `@CrossOrigin(origins = "*")`
- **State graph**: defined in `DataAgentConfiguration.nl2sqlGraph()` using `com.alibaba.cloud.ai.graph.StateGraph`. Nodes in `workflow/node/`, edge dispatchers in `workflow/dispatcher/`.
- **Workflow**: IntentRecognition → EvidenceRecall → QueryEnhance → SchemaRecall → TableRelation → FeasibilityAssessment → Planner → PlanExecutor → [SqlGenerate→SemanticConsistency→SqlExecute | PythonGenerate→PythonExecute→PythonAnalyze | ReportGenerator | HumanFeedback]
- **Dual-package pattern**: local code under `com.phoenix.data.*` references upstream types from `com.alibaba.cloud.ai.dataagent.*` (provided by `spring-ai-alibaba-graph-core` JAR).
- **Persistence split**: `phoenix-data` uses plain MyBatis (`mybatis-spring-boot-starter`). `phoenix-agent-core` uses MyBatis-Flex (`mybatis-flex-spring-boot4-starter`) with mappers extending `BaseMapper<T>`. Entity annotations (`@Table`, `@Id`, `@KeyType`) are from `com.mybatisflex.*` in both.

## Database

- Schema SQL: `sql/all_schema.sql` (PostgreSQL syntax with `GENERATED BY DEFAULT AS IDENTITY`, `JSONB`, `COMMENT ON`)
- Vector store: `PgVectorStore` by default (configured in `DataAgentConfiguration`), not in-memory SimpleVectorStore.
- Supported dialects: MySQL, PostgreSQL, Oracle, SQL Server, H2, Hive, Dameng (each with `DBAccessor` + `JdbcConnectionPool` + `JdbcDdl` implementations).

## Framework Quirks

- Lombok `@AllArgsConstructor` is used extensively for constructor injection.
- `springdoc-openapi` deps are **commented out** in POM.
- Netty native transports: `netty-resolver-dns-native-macos` (osx-x86_64), `netty-transport-native-epoll` (linux-x86_64).
- `hive-jdbc` has aggressive exclusions for Jetty, Hadoop, Tomcat, Servlet to avoid conflicts.
- Declared but **not wired** in POM: maven-checkstyle-plugin, spotless-maven-plugin, jacoco-maven-plugin.
- Sa-Token integration for auth (`sa-token-reactor-spring-boot4-starter`).
- OpenTelemetry → Langfuse tracing configured.
